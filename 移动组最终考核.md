# 移动组最终考核

相信大家经过寒假训练营和开学后的小组培训后已经对移动开发有了一定的了解，现在是时候检验学习 成果了。你们将用所学的知识完成一个较为完整的APP，我们会结合前期的综合表现以及最终考核的完成 程度来挑选出适合的学弟学妹加入QG工作室这个大家庭。

## 考核题目：

背景：大三的小明是计算机学院一个勤奋的“图书馆钉子户”，但他常因为沉迷学习忘记看天气预报而淋 雨。同时，面对海量的技术知识（如 OkHttp、JVM），他感到碎片化的心得难以沉淀，系统的学习规划 也总被打乱。请你为小明开发这款 APP，助他成长为一名能够高效学习、不断总结提升的优秀程序员。

注：此题目只是给一个大方向，细节的实现可以各自发挥。

## 基本要求（必须完成）：

1. 登录注册模块

使用sqlite数据库储存数据（参数自定）

2. 主架构

使用ViewPager实现页面的滑动切换，至少三个页面

3. 天气模块

使用网络请求接入天气API，显示当前地址与天气

API网站：http://tianqiapi.com/

4. 心得沉淀模块

使用列表展示标题

新建：用户可输入标题和正文

修改：用户可进入详情页可以修改标题或正文

删除：用户可在列表中可以选中单条或者多条记录进行删除

5. 技能规划模块

列表展示

新建：用户可选择开始时间段（年月日时分）、学习持续时间与具体要学习的技能（标题）， 学习规划（正文）

删除：用户可在列表中可以选中单条或者多条记录进行删除

修改：用户可修改开始时间段，持续时间，标题与正文

查询：用户可以通过年月日或者年月查询某一天或者某一个月的全部计划。

6. 界面尽量美观，运行要流畅不能有明显卡顿，运行过程中不能闪退，要有一定的错误反馈（错误反 馈是指当用户操作有误时通过特定方式提醒用户，比如没有输入密码就点登录应该给一个Toast“密 码不能为空”作为提示）

## 进阶要求（不强制，但是完成越多得分越高）

1. AI 智能助手（深度集成）：

基础问答：另开一个 AI 页面做成问答大模型的形式，支持单轮对话做成聊天样式显示即可， 有能力可尝试实现有上下文的对话

心得提炼：在心得详情页，增加一个“AI 总结”按钮，调用大模型 SDK 生成这段心得或是碎碎 念的核心关键词、情感摘要以及一些建议。

计划拆解：在新建规划时，用户输入技能名称，询问AI可给出具体的学习路径建议，并自动填 入正文详情中。

2. 可视化成长：

将本周/本月的学习时长以柱状图或折线图展示，建议利用 MPAndroidChart 库，。

对已完成的规划和未完成的有所区分，尝试利用spinner

3. 智能交互：

精准触达：当技能规划的开始时间到达时，通过 Notification 或自定义弹窗对用户做出提示。

便捷登录：支持“记住账号密码”功能。

更新时间：心得详情页显示最后一次修改的时间（年月日时分）

及时反馈：发送网络请求时可能会有延迟，如何告诉用户目前正在等待回调而不是发送失败

4. 可更换软件的主题颜色，如何搭配更能得到用户的肯定

5. 自己发挥想象力增加需求完善软件

## Tips

1. 大模型可选星火的，难度比较合适

https://www.xfyun.cn/doc/spark/AndroidSDK.html#_1-sdk%E4%BB%8B%E7%BB%8D

https://blog.csdn.net/weixin_56649281/article/details/136569427

2. 项目结构要清晰，写好代码注释方便后续的拓展和维护

3. 鼓励使用小组培训中讲的MVP架构

4. App中的各种图标建议使用SVG矢量图，下面是推荐的矢量图网站

https://www.svgrepo.com/

5. 代码提交到github仓库时注意分支管理，不要只在main分支上进行开发

6. 不要使用未经允许的依赖，如有疑问可以向师兄咨询

7. 尝试自己阅读报错信息并且解决bug，不会可以寻求ai的帮助，至少比一遇到不会的就想着去问师 兄好。
